# Electricity generation planning model

A [Shiny for Python](https://shiny.posit.co/py/) app that models the cost and
carbon emissions of scenarios which optimize the mix of electricity generation
resources (wind, solar, batteries, and gas) needed to serve load. The loads are
based on the forecasted requirements of four northern Colorado communities.

Under the hood it solves an 8,760-hour (one full year) linear program with
[OR-Tools](https://developers.google.com/optimization) GLOP. Users can vary the
allowable capacity and cost of each resource, optimize for either cost or carbon
emissions, and run stress tests of limited renewable availability.

## What you can do

- Adjust the allowable range of installed capacity, resource costs, gas limits,
  and lifecycle carbon value in the sidebar.
- Click **create run** to solve. The optimization runs in the background (~15–60s
  depending on inputs) so the interface stays responsive.
- Review resource capacities, cost metrics, and generation metrics as cards, plus
  an interactive hourly load-and-generation plot.
- **Save**, **delete**, and **download** runs (all inputs and results as CSVs in a
  zip) from the sidebar.

## Project layout

```
app.py            # Shiny entrypoint (root — required by Posit Connect Cloud)
pyproject.toml    # dependency source of truth (managed with uv)
uv.lock
requirements.txt  # GENERATED from uv — Connect Cloud installs from this; do not edit
manifest.json     # GENERATED by rsconnect — describes the app for Connect Cloud
.python-version   # 3.11 (matches the Posit Connect Cloud runtime)
src/              # LP.py model, parameters.py defaults/cost formulas, db.py store, utils.py plot, profiles.csv

```

## Local development

Requires [uv](https://docs.astral.sh/uv/) and Python 3.11 (uv will fetch it).
The project pins to 3.11 because that is the Python runtime Posit Connect Cloud
provisions, so local, Docker, and deploy environments match.

```bash
uv sync                     # create the .venv from uv.lock
uv run shiny run app.py     # serve at http://127.0.0.1:8000 (add --reload while editing)
```

### Managing dependencies

`pyproject.toml` + `uv.lock` are the source of truth. `requirements.txt` and
`manifest.json` are generated artifacts for Posit Connect Cloud — after any
dependency change, re-lock and regenerate them:

```bash
uv lock
uv run python scripts/update_manifest.py
```

## Testing

```bash
uv run pytest -m "not slow and not e2e"   # fast unit tests (< 1s)
uv run pytest -m slow                      # full LP solve integration tests (~15s)
uv run playwright install chromium         # one-time, for e2e
uv run pytest -m e2e                        # browser end-to-end tests
uv run pytest                              # everything
```

E2E tests follow the
[Shiny end-to-end testing guide](https://shiny.posit.co/py/docs/end-to-end-testing.html).

## Deploy to Posit Connect Cloud

The app is structured for [Posit Connect Cloud](https://connect.posit.cloud/):
`app.py` at the root, dependencies in `requirements.txt`, and a `manifest.json`.

1. Regenerate the deploy artifacts whenever the app or dependencies change
   (re-exports `requirements.txt` and writes a trimmed `manifest.json` —
   rsconnect bundles every tracked file otherwise, so the script excludes
   dev tooling, docs, and the reference PDFs):
   ```bash
   uv run python scripts/update_manifest.py
   ```
2. Commit and push to GitHub.
3. In Posit Connect Cloud, create new content from the GitHub repo. It detects the
   Shiny `app.py` and installs from `requirements.txt`.

> **Note:** Saved runs live in a per-session in-memory SQLite database and the
> download zip is written to ephemeral working-directory storage, so runs are not
> persisted across sessions — the same behavior as running locally.
